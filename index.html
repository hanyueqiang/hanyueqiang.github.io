退出登录删除cookie信息

发表于 2018-01-16

什么是cookie

Cookie 是一些数据, 存储于你电脑上的文本文件中。
当 web 服务器向浏览器发送 web 页面时，在连接关闭后，服务端不会记录用户的信息。Cookie 的作用就是用于解决 “如何记录客户端的用户信息”

设置 cookie 值的函数

创建一个函数用于存储访问者的名字：

function setCookie(cname,cvalue,exdays)
{
      var d = new Date();
      d.setTime(d.getTime()+(exdays*24*60*60*1000));
      var expires = "expires="+d.toGMTString();
      document.cookie = cname + "=" + cvalue + "; " + expires;
}

函数解析：

以上的函数参数中，cookie 的名称为 cname，cookie 的值为 cvalue，并设置了 cookie 的过期时间 expires。
该函数设置了 cookie 名、cookie 值、cookie过期时间。

获取 cookie 值的函数

function getCookie(cname)
{
  var name = cname + "=";
  var ca = document.cookie.split(';');
  for(var i=0; i<ca.length; i++) 
  {
    var c = ca[i].trim();
    if (c.indexOf(name)==0) return c.substring(name.length,c.length);
  }
  return "";
}

//two
function getCookie(name)
  {
    var arr,reg=new RegExp("(^| )"+name+"=([^;]*)(;|$)");
    if(arr=document.cookie.match(reg))
      return (arr[2]);
    else
      return null;
  }

函数解析：

cookie 名的参数为 cname。
创建一个文本变量用于检索指定 cookie :cname + “=”。
使用分号来分割 document.cookie 字符串，并将分割后的字符串数组赋值给 ca (ca = document.cookie.split(‘;’))。
循环 ca 数组 (i=0;i<ca.length;i++)，然后读取数组中的每个值，并去除前后空格 (c=ca[i].trim())。
如果找到 cookie(c.indexOf(name) == 0)，返回 cookie 的值 (c.substring(name.length,c.length)。
如果没有找到 cookie, 返回 “”。

gitReset探索之路

发表于 2017-10-16

利用闲暇时间整理下最近工作中遇到的问题，前几天同事问我他的代码commit过后没有先拉代码直接push到远程了，这种情况我以前也干过，当时用此命令解决的这个问题，所以我没有思索的跟他说用git reset --hard {id}来解决；结果就是遇到大麻烦；git reset --hard 命令解释：强制返回到某次提交前的源码状态；用此命令一定要慎重！再慎重！以下是git reset时几种情况：

一.commit了全部修改内容 使用git reset --hard 后情况

下图是我新建一个文件，修改了两个文件

然后add commit 后git log 查看我commit的记录new hhh.js

我们使用git reset --hard 2460d... ，即使我们在这之前push或者拉代码都会回退到这个版本之前，但是commit的代码依然存在，所以这个没有问题；

二.commit了部分代码，另一部分add了但没有commit

我只commit一个文件，add了另外两个文件；

我们使用git reset --hard {id} 过后我们add的东西不见了，是不挺着急，这时候你用git reflog 查看版本记录再次回滚到commit记录下，发现没鸟用；这时候我们可以执行git fsck --lost-found 命令，然后你就可以在本地项目文件中路径为.git/lost-found/other中找到它们，你会发现你add的文件竟然藏在那里，是不是感觉棒棒哒！也可以通过find .git/objects -type f | xargs ls -lt | sed 30q这个命令，你就可以找到最近被你add到本地仓库的30个文件（切记不要太多，你会晕的；

三.commit了部分代码，另一部分没有add；

执行git reset --hard后等同于执行delete命令一样（除非你能逆天）；这也是我同事遇到情况，但幸好他用VS Code编辑器有历史对比救了他一命，不然真的要从新写了！如果有哪位大神对这种情况有好的解决办法，求指教！

最后，一定要养成commit习惯！即使不commit也要多git add几次也无妨！

hexo介绍

hexo是基于nodejs开发的一个软件,也成为工具,所以我们必须要有一个nodejs环境,首先打开git Bash Here输入node -v 确定你的电脑是否安装nodejs,然后验证npm -v ,没有的话先安装nodejs.

创建步骤

  • 在任意一个盘下打开git Bash Here,执行命令hexo,hexo就是用来做博客的,他路径不能随便乱写.选好路径后(比如在E盘下)创建一个文件夹,怎么创建呢?不能直接新建文件夹,要用git Bash Here(以后简称git命令行) 输入hexo init blog(文件命名自己随便写)没网的情况下也可以创建,自身会克隆.在git命令行输入ls 看下有没有这个文件夹.创建好后执行在git命令行输入 cd blog/ 进入blog文件内,打开git命令行.
