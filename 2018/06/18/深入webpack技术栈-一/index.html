一.对webpack理解

是一个前端资源加载/打包工具。它将根据模块的依赖关系进行静态分析，然后将这些模块按照指定的规则生成对应的静态资源。它做的事情是，分析你的项目结构，找到JavaScript模块以及其它的一些浏览器不能直接运行的拓展语言（Scss，TypeScript等），并将其转换和打包为合适的格式供浏览器使用。可以将多种静态资源 js、css、less 转换成一个静态文件，减少了页面的请求。

注意

在webpack v3.x 版本使用
webpack hello.js hello.min.js
会把hello.js 默认打包成文件hello.min.js
但是在当前v4.x版本会报错，显示没有mode项，默认有production和development两种，我们输入
webpack --mode development 依然有错，这是因为webpack4.x以目录下./src/index.js作为入口，需要新建src目录hello.js改为index.js,同时webpack4.x不支持webpack 文件a 文件b打包方式，而是直接运行webpack –mode development 或者 webpack –mode production,这样会默认打包，入口文件是./src/index.js,但是这样会很麻烦，我们在package.json中配置一下

这样每次执行npm run dev

在处理css文件需要依赖loader
npm install css-loader style-loader --save-dev
css-loader : 使webpack可以处理css文件

style-loader : 新建一个css标签,使css-loader 处理过的文件放进去，然后插入html标签中
安装后可以配置到webpack命令中 也可以手动添加

在命令行：
webpack --mode development --module-bind 'css=style-loader!css-loader' --watch

–watch: 监听，每次保存自动执行

还有一些参数有需要配置
webpack --mode development --module-bind 'css=style-loader!css-loader' --progress --display-modules --display-resons

–progress:出现有进度条打包过程

– display-modules: 会把所有模块列出来

–diplay-resons : 列出法打包原因
